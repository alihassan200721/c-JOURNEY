Short-Circuit Evaluation
C &&, || — order matters
In C, && and || short-circuit — the right side may not be evaluated.
A && B: if A is false, B is never evaluated A || B: if A is true, B is never evaluated
// Safe null check — ptr && is evaluated first if (ptr != NULL && ptr->value > 0) { } // If ptr is NULL, ptr->value never runs // This is the mathematical guarantee of &&
This is not just a language feature — it is a consequence of the mathematical definition of AND.
Karnaugh Maps (K-Maps)
Logic simplifying conditions
K-maps let you find the simplest Boolean expression for a truth table. Used in hardware design and complex conditional simplification.
AB 00 01 11 10 0 | 1 | 0 | 1 | 1 | ← C=0 1 | 0 | 1 | 1 | 0 | ← C=1 Group the 1s into rectangles (size = power of 2). Each rectangle → one AND term. Result = OR of all rectangles.
In code: if you have 4+ nested conditions, draw the truth table and simplify with a K-map before writing the if-else.
Chapter V
Linear Algebra for 3D & Graphics

As a 3D artist working in Blender and building product visualisations, this is the mathematics living inside every transform, camera, and shader. Knowing it lets you write your own renderers and understand exactly what Blender is computing.

Vectors
3D positions, directions, normals
A vector is a magnitude and direction. In 3D, a point in space is just a vector from the origin.
Basic operations: v = (x, y, z) |v| = √(x² + y² + z²) ← magnitude (length) v̂ = v / |v| ← unit vector (normalise) a · b = axbx + ayby + azbz ← dot product (scalar) a × b = (matrix det) ← cross product (vector, ⊥ to both)
typedef struct { float x,y,z; } Vec3; float dot(Vec3 a, Vec3 b) { return a.x*b.x + a.y*b.y + a.z*b.z; } // dot == 0 → perpendicular // dot == |a||b| → parallel, same dir // dot == -|a||b| → antiparallel
Matrices & Transforms
3D rotation, scale, projection
Every transform in 3D (move, rotate, scale) is a matrix multiplication. The 4×4 homogeneous matrix is the workhorse of all realtime graphics.
Translation matrix (move by dx, dy, dz): [1 0 0 dx] [0 1 0 dy] [0 0 1 dz] [0 0 0 1]
Compose transforms by multiplying matrices: M_final = M_projection × M_view × M_model
Matrix multiplication is NOT commutative: A×B ≠ B×A. Order always matters in 3D.
Dot Product — The Angle Test
3D lighting, culling, normals
a · b = |a| |b| cos θ
Lighting (Lambertian): how much light hits a surface? intensity = max(0, dot(normal, light_dir)) → normal · light = 1.0 → fully lit (face-on) → normal · light = 0.0 → grazing angle → normal · light < 0 → back face (cull it)
float light(Vec3 N, Vec3 L) { // N and L must be unit vectors float d = dot(N, L); return d > 0 ? d : 0.0f; }
Cross Product — The Normal Maker
3D surface normals, torque, area
The cross product of two vectors gives a third vector perpendicular to both — this is how surface normals are computed from triangle vertices.
a × b = (formula from 3×3 determinant): x = ay×bz − az×by y = az×bx − ax×bz z = ax×by − ay×bx
Area of parallelogram / winding order test: |a × b| = |a| |b| sin θ sign of z-component → clockwise vs counterclockwise
Quaternions — Rotation Without Gimbal Lock
3D animations, cameras, rotations
Euler angles (roll, pitch, yaw) suffer from gimbal lock — a situation where two axes align and you lose a degree of freedom. Blender, game engines, and animation systems use quaternions to avoid this.
A quaternion q = w + xi + yj + zk i² = j² = k² = ijk = −1 |q| = √(w²+x²+y²+z²) = 1 (unit quat)
To rotate vector v by quaternion q:
v' = q × v × q⁻¹
Axis-angle → quaternion (rotate θ around axis n̂): w = cos(θ/2) x = nx × sin(θ/2) y = ny × sin(θ/2) z = nz × sin(θ/2)
SLERP — smooth interpolation between rotations: slerp(q1, q2, t) = q1(q1⁻¹q2)^t Used in every keyframe animation system.
Chapter VI
Discrete Mathematics

Discrete math is the language of algorithms. Sets, graphs, combinatorics, and recursion — these are the structures your data lives in and the tools you use to reason about correctness.

Set Theory & Relations
Algorithms hash sets, databases, logic
Operations: A ∪ B = union (elements in A or B or both) A ∩ B = intersection (elements in both) A \ B = difference (in A, not in B) A ⊆ B = A is a subset of B |A| = cardinality (number of elements)
Hash sets, maps, and databases are built on set theory. SQL JOIN is set intersection. UNION query is set union.
Inclusion-exclusion (overlapping sets): |A ∪ B| = |A| + |B| − |A ∩ B|
Graph Theory
Algorithms BFS, DFS, Dijkstra, trees
A graph G = (V, E) — vertices (nodes) and edges. Most real-world problems map to graphs: networks, dependencies, pathfinding, state machines.
Key properties: Degree of vertex v = number of edges at v Tree = connected graph with |E| = |V| − 1, no cycles DAG = Directed Acyclic Graph (build systems, Git) Bipartite = vertices split into 2 sets, edges only between sets
Euler's formula for planar graphs: V − E + F = 2
where F = number of faces (including outer face).
Recursion & Recurrences
Algorithms divide & conquer, trees
Recursive algorithms have mathematical recurrence relations that define their complexity.
Merge sort recurrence: T(n) = 2T(n/2) + O(n) → solves to O(n log n)
Master Theorem — solving T(n) = aT(n/b) + f(n): Case 1: f(n) = O(n^(log_b a − ε)) → T = O(n^(log_b a)) Case 2: f(n) = Θ(n^(log_b a)) → T = O(n^(log_b a) log n) Case 3: f(n) = Ω(n^(log_b a + ε)) → T = O(f(n))
Combinatorics
Algorithms permutations, probability, hashing
Counting arrangements — order matters (permutations): P(n,k) = n! / (n−k)! ← k items from n, ordered
Counting selections — order doesn't matter (combinations): C(n,k) = n! / (k!(n−k)!) ← "n choose k"
Birthday paradox (collision probability): With 23 people: P(shared birthday) > 50% → hash collision chance is higher than intuition suggests → this is why hash tables need good collision handling
Chapter VII
Probability, Statistics & ML Math

For your machine learning and F1 prediction work — this is the foundation. Every model is applied probability theory. Every loss function is a statistical concept.

Probability Fundamentals
ML Bayes, distributions, prediction
Bayes' theorem — update beliefs with evidence: P(A|B) = P(B|A) × P(A) / P(B) P(A) = prior probability (before evidence) P(B|A) = likelihood (how probable is the evidence if A?) P(A|B) = posterior (updated belief)
Law of Total Probability: P(B) = Σ P(B|Aᵢ) × P(Aᵢ)
Every naive Bayes classifier, every spam filter, every F1 outcome predictor — Bayes all the way down.

Linear Regression
ML prediction, curve fitting
The simplest model: fit a line through data to predict future values.
ŷ = w₁x + w₀
Least squares — minimise sum of squared errors: L(w) = Σ (yᵢ − ŷᵢ)² Optimal w₁ = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)² Optimal w₀ = ȳ − w₁x̄
Gradient descent (how neural networks learn): move w in the direction of −∇L. The gradient ∇L is just the vector of partial derivatives — calculus applied to optimisation.
Information Theory
ML compression, decision trees, entropy
Entropy — measure of uncertainty/information: H(X) = −Σ P(xᵢ) log₂ P(xᵢ) H = 0 bits → perfectly predictable (certain) H = 1 bit → fair coin flip H = high → high uncertainty, more info needed
Cross-entropy loss (used to train classifiers): L = −Σ yᵢ × log(ŷᵢ)
Decision trees split on the feature that reduces entropy the most (information gain).
Chapter VIII
Calculus Essentials for Programmers

You don't need to solve integrals by hand. You need to understand what a derivative means and why gradient descent works. These concepts are the engine of modern ML.

Derivatives — The Rate of Change
ML gradient descent, backprop
f'(x) = lim[h→0] (f(x+h) − f(x)) / h
Common derivatives to know: d/dx [xⁿ] = n·xⁿ⁻¹ (power rule) d/dx [eˣ] = eˣ (exponential) d/dx [ln x] = 1/x (natural log) d/dx [sin x] = cos x d/dx [f(g(x))] = f'(g(x))·g'(x) (chain rule)
The chain rule is literally how backpropagation in neural networks is derived — it's the rule for differentiating through nested functions.
Gradient & Gradient Descent
ML training neural networks
The gradient ∇f is the vector of all partial derivatives. It points in the direction of steepest ascent. Move in the opposite direction to minimise.
Gradient descent update rule: w = w − α × ∇L(w) w = weights, α = learning rate, L = loss function
# In Python — one gradient descent step for param in model.parameters(): param -= learning_rate * param.grad # param.grad is ∂L/∂param
Learning rate α too large → overshoot, diverge. Too small → slow, might get stuck in local minima.
Integrals & Area Under Curve
3D ML rendering, ROC curves
∫ f(x) dx = area under f(x)
In 3D rendering — integrating the rendering equation over all light directions gives you the pixel colour. In ML — AUC (area under ROC curve) measures classifier quality.
Fundamental theorem of calculus: ∫[a→b] f(x) dx = F(b) − F(a) where F'(x) = f(x)
Monte Carlo integration (how path tracers work): ∫ f(x) dx ≈ (1/N) Σ f(xᵢ) Sample random points, average the values. More samples → more accurate result.

Chapter I
Number Systems & Binary Arithmetic

The machine speaks in binary. Everything else — decimal, hex, octal — is a human convenience layered on top. Understanding this is prerequisite to understanding memory.

Positional Number Systems
C integer literals, bit ops
Every number system is just base-N counting. Each digit position is a power of the base.
Decimal (base 10) 1024 = 1×10³ + 0×10² + 2×10¹ + 4×10⁰
Binary (base 2) — what the CPU sees 1010 = 1×2³ + 0×2² + 1×2¹ + 0×2⁰ = 10
Hexadecimal (base 16) — compact binary notation 0xFF = 15×16¹ + 15×16⁰ = 255 0xDEAD = 57005 decimal
In C, prefix 0b = binary, 0x = hex, 0 = octal.
Two's Complement
C signed integers, overflow
How negative numbers are stored. Not "add a minus sign" — invert all bits, add 1.
8-bit example: represent −5 +5 = 0000 0101 flip= 1111 1010 +1 = 1111 1011 ← this is −5 in two's complement
Why this matters: int wraps on overflow (UB in C!). INT_MAX + 1 doesn't give you a big number, it gives you INT_MIN. The MSB (leftmost bit) is the sign bit.
Range formula: for N-bit signed: −2N−1 to 2N−1−1. For 32-bit int: −2,147,483,648 to 2,147,483,647.

IEEE 754 Floating Point
C float, double precision
float is not a real number. It is an approximation. The 32-bit layout:
┌─┬────────┬───────────────────────┐ │S│ Exp(8) │ Mantissa (23 bits) │ └─┴────────┴───────────────────────┘ 1 8 23 value = (−1)^S × 1.mantissa × 2^(exp−127)
This is why 0.1 + 0.2 ≠ 0.3 in any language. Never compare floats with ==. Use epsilon comparison:
#define EPSILON 1e-9 fabs(a - b) < EPSILON // "equal enough"
Chapter II
Modular Arithmetic & Integer Math

The modulo operation is one of the most used tools in programming — hashing, circular buffers, day-of-week, cryptography. Understand it deeply.

Modulo — The Clock Model
C %, loops, hashing, wrapping
Think of modulo as a clock. After 12 comes 1, not 13. After N comes 0.
a mod n = remainder when a ÷ n
// Circular buffer — wrap index index = (index + 1) % BUFFER_SIZE; // Even/odd check if (n % 2 == 0) { /* even */ } // Hash table bucket bucket = hash(key) % TABLE_SIZE; // Day of week (0=Sun) after N days day = (current_day + N) % 7;
Warning: in C, % on negative numbers can give negative results. Use ((a % n) + n) % n for always-positive modulo.
Integer Division & Floor/Ceil
C pagination, chunking, alignment
Integer division in C truncates toward zero. This is not the same as floor for negatives.
Ceiling division (pages needed to hold N items in groups of K): ceil(N / K) = (N + K − 1) / K ← integer, no float needed
// How many 4-byte words to store N bytes? int words = (N + 3) / 4; // Align address to 16-byte boundary size_t aligned = (addr + 15) & ~15; // same as: (addr + 15) / 16 * 16 // Number of pages for N bytes (page=4096) int pages = (N + 4095) / 4096;

Algorithms number theory basics
